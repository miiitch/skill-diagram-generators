# Contribution Guidelines

Thank you for your interest in contributing to **skill-diagram-generators**! 

## Getting Started

1. **Fork** the repository
2. **Clone** your fork locally
3. Create a **feature branch** for your changes: `git checkout -b feature/your-feature-name`
4. **Test** your changes thoroughly
5. **Commit** with clear, descriptive messages
6. **Push** to your fork and submit a **Pull Request**

## Code Style

- Use clear, descriptive variable names
- Add comments for complex logic
- Keep lines reasonably short and readable
- Follow the existing code structure in the repository

## Testing

Before submitting a PR:
- Test your changes on at least one OS (Windows, macOS, or Linux)
- Include screenshots/output if your changes affect diagram generation
- Verify that existing functionality still works

## Reporting Issues

When reporting bugs, please include:
- Your OS and software versions
- Clear steps to reproduce
- Expected vs actual behavior
- Screenshots or error logs if applicable

## Questions?

Feel free to open an **issue** to ask questions or discuss ideas before implementing major changes.
